# E2E 테스트 체크리스트

## 테스트 개요

이 문서는 쇼핑몰 MVP의 주요 사용자 플로우를 점검하기 위한 체크리스트입니다.

## 테스트 환경

- **브라우저**: Chrome, Firefox, Safari, Edge (최신 버전)
- **디바이스**: 데스크톱, 태블릿, 모바일
- **모드**: 라이트 모드, 다크 모드

---

## 1. 회원가입/로그인 플로우

### 1.1 회원가입
- [ ] 회원가입 페이지 접근 가능 (`/sign-up`)
- [ ] 이메일 입력 필드 표시
- [ ] 비밀번호 입력 필드 표시
- [ ] 회원가입 버튼 클릭 시 폼 검증 동작
- [ ] 유효한 정보 입력 시 회원가입 성공
- [ ] 회원가입 후 자동 로그인 및 리다이렉트
- [ ] Supabase `users` 테이블에 사용자 정보 저장 확인

### 1.2 로그인
- [ ] 로그인 페이지 접근 가능 (`/sign-in`)
- [ ] 이메일/비밀번호 입력 필드 표시
- [ ] 로그인 버튼 클릭 시 폼 검증 동작
- [ ] 유효한 정보 입력 시 로그인 성공
- [ ] 로그인 후 홈페이지로 리다이렉트
- [ ] 네비게이션에 사용자 정보 표시 (UserButton)

### 1.3 로그아웃
- [ ] 로그아웃 버튼 클릭 시 로그아웃 성공
- [ ] 로그아웃 후 인증 보호 페이지 접근 불가
- [ ] 세션 정리 확인

## 2. 상품 탐색 플로우

- [ ] 카테고리 섹션 표시
- [ ] 인기 상품 목록 표시 (최대 8개)

- [ ] 페이지네이션 동작 확인
- [ ] 정렬 기능 동작 (가격 오름차순/내림차순)
- [ ] 카테고리 필터 동작
- [ ] 상품 카드 클릭 시 상품 상세 페이지로 이동

### 2.3 상품 상세 페이지

- [ ] 장바구니 담기 버튼 표시

- [ ] 존재하지 않는 상품 접근 시 404 페이지 표시
- [ ] 뒤로가기 버튼 동작

## 4. 주문 플로우

### 4.1 배송지 입력
- [ ] 체크아웃 페이지 접근 가능 (`/checkout`)
- [ ] 로그인하지 않은 경우 로그인 페이지로 리다이렉트
- [ ] 장바구니가 비어있으면 장바구니 페이지로 리다이렉트
- [ ] 배송지 입력 폼 표시
- [ ] 필수 필드 검증 동작
- [ ] 주문 요약 표시 (상품 목록, 총 금액)
- [ ] 폼 제출 시 주문 생성

### 4.2 결제
- [ ] 주문 생성 후 결제위젯 표시
- [ ] Toss Payments 위젯 로딩 확인
- [ ] 결제 수단 선택 가능
- [ ] 결제 금액 정확성 확인
- [ ] 결제 요청 시 Toss Payments 페이지로 이동
- [ ] 테스트 결제 진행 (카드번호: 4242 4242 4242 4242)
- [ ] 결제 성공 시 성공 페이지로 리다이렉트
- [ ] 결제 실패 시 실패 페이지로 리다이렉트

### 4.3 주문 완료
- [ ] 결제 성공 후 주문 상태 업데이트 (`pending` → `confirmed`)
- [ ] 주문 완료 페이지 표시 (`/orders/[id]`)
- [ ] 주문 정보 표시 (주문번호, 상품 목록, 배송지, 총 금액)
- [ ] 주문 상태 배지 표시
- [ ] 장바구니 비우기 확인

## 참고사항

- 모든 테스트는 실제 사용자 시나리오를 기반으로 수행
- 각 플로우는 처음부터 끝까지 연속적으로 테스트
- 다양한 브라우저 및 디바이스에서 테스트 권장
- 프로덕션 환경과 유사한 환경에서 테스트 권장
